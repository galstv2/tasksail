# Scratch Space

This directory is reserved for temporary working materials created during planning,
analysis, prototyping, and other short-lived implementation support tasks.

Typical uses include:

- draft specifications
- exploratory notes
- temporary refactor plans
- disposable research artifacts
- intermediate working files that should not become part of the repository's
  long-term source of truth

Content in this directory should be treated as non-authoritative and safe to
delete when it is no longer useful.

Repository guidance:

- files under `scratchspace/` are always ignored by git
- this README is kept tracked so the purpose of the directory remains clear
- durable project documentation, implementation artifacts, and workflow records
  should live in their proper repository locations instead of this directory
